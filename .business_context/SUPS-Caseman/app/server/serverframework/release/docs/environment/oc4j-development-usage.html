Installing OC4J (oc4j-9.0.4.zip)
Unzip ServerFramework\release\oc4j-9.0.4.zip into <OC4J_HOME> this will result in the directory structure Note: This oc4j is fully self contrained and has java extensions from j2sdk 1.4.1_07 in the /java-ext directory

The main directories after installation are <OC4J_HOME>, <OC4J_HOME>\j2ee\home\conf
and <OC4J_HOME>\j2ee\home

edit <OC4J_HOME>\j2ee\home\application.xml and add the line
<library path="C:/j2sdk1.4.2_07/lib/tools.jar"/>
after the other lines <library path="...."/>

http://localhost:7070/examples/jsp/dates/date.jsp (this will test your tools.jar setting above)
or go to http://localhost:7070 click JSP, click execute on one of the Execute links

    java -jar ${oc4j.home}/admin.jar ${oc4j.deploy.ormi} ${oc4j.deploy.username} ${oc4j.deploy.password} -deploy -file ${dist.dir}/${dist.name} -deploymentName ${module.name}

    Extras on about your OC4J Installation
    http://localhost:7070/dmsoc4j/Spy
    For your OC4J standalone instance, there are metrics available using the Spy servlet. The Spy servlet is part of the DMS (Dynamic Monitoring Service, ) application which is deployed by default into the default web application.
    For your OC4J standalone instance, there are metrics available using the Spy servlet. The Spy servlet is part of the DMS (Dynamic Monitoring Service, a subsystem to collect and display runtime data for various aspects of the product and applications that it is running ) application which is deployed by default into the default web application.

    "Spy is a pre-packaged servlet that reports performance metrics for OC4J standalone. Spy reports performance data for an OC4J process." Resetting DMS Data To reset the performance data, you send a URL to the Spy servlet to inform it that it should  do a reset.  This basically involves tacking on an "operation=reset" parameter onto a Spy URL.
    To reset the performance data, you send a URL to the Spy servlet to inform it that it should  do a reset.  This basically involves tacking on an "operation=reset" parameter onto a Spy URL.

    The basic form of the URL to perform a reset looks likes:

    "http://localhost:7070/dms/Spy?operation=reset&name=/name1&recurse=all"
