RELEASE NOTE FOR SUPS FRAMEWORK 9.0.7

STATUS: Released
------------------------------------

A new Servlet has been added to the invoker application CheckServlet

Accessed with:
	http://w2nzsh9201:7070/${application.name}_invoker/FrameworkCheckServlet?user=dzmy2y&pass=Password01
	
	This servlet should be access over https in the non development environments.
	
	user and pass are optional parameters.
	
	This servlet returns the contents of the following files:
	
	framework.version.properties which is generated during distribution
	
	and version.properties which can/should be generated by the product containing the same properties as found 
	in the ServerFramework/release/default_conf directory
		
the ConfigUtil class has a default constructor which gives the project team access to the path of their
project_config.xml path as welll as the applicationName which exists in this file

ConfigUtil config = new ConfigUtil();
config.getApplicationName(); //returns the name the application is deployed as ie: casemanpt01

On deployment the project_config.xml will have the following configuration items added and they will 
default to the application:name configuration item if not configured by the application

	<item id="application:serviceprefix" class="uk.gov.dca.db.impl.StringConfigurationItem">/test_project</item>
	<item id="application:webprefix" class="uk.gov.dca.db.impl.StringConfigurationItem">/sample-apps</item>

	<!-- set this to the same as your <services baseURL in the client applicationconfig.xml -->
	<item id="application:serviceprefix" class="uk.gov.dca.db.impl.StringConfigurationItem">/ediary</item>

	<!-- set this to the same as application-config baseURL="/familyman" in the client applicationconfig.xml -->
	<item id="application:webprefix" class="uk.gov.dca.db.impl.StringConfigurationItem">/ediary</item>

Enhancements
	ContentServlet rewrites Framework Exceptions
	
Removed Validator Log component from sups_config.xml
	<validator name="Log" class="uk.gov.dca.pipeline.LogComponent"/>

RELEASE NOTE FOR SUPS FRAMEWORK 9.0

STATUS: released

List of significant changes
---------------------------

1) D304 - All SQL queries logged at level "INFO" when they are executed by the database mapping framework.

2) D699 - JDOM documant of input parameters now provided to PrintersListDAO.

3) D710 - ReportInvoker failed for list report requests.

4) E119 - "SecurityBean.getUserDetails" updated so can optionally include the home court in
the returned XML. Example input:

<params>
	<param name='userId'>id1</param>
	<param name='getHomeCourt'>true</param>
</params>

Returns:

<UserPersonalDetails>
	<HomeCourt/>
	<UserId/>
	<Forenames/>
	<Surname/>
	<Title/>
	<UserShortName/>
	<JobTitle/>
	<Extension/>
	<IsActive/>
	<StyleProfile/>
	<DefaultPrinter/>
</UserPersonalDetails>

If parameter "getHomeCourt" is not provided or is not true then the home court is not fetched.
The method "updateUser" is unchanged. In summary, its input is of the form

<params>
	<param name='userId'/>
	<param name='courtId'/>
	<!-- following params are all optional: -->
	<param name='activeUserFlag'/>
	<param name='styleProfile'/>
	<param name='accessSecurityLevel'/>
	<param name='title'/>
	<param name='jobTitle'/>
	<param name='extension'/>
	<param name='userShortName'/>
</params>
