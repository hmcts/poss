callService

<static> function callService(mappingName, parameters, handler, async, showProgress)
    Call Server Side service The handler is an object which is called to process callbacks resulting from this service call. The onSuccess method is called when the service returns without error. This method must be implemented. The handler object may also implement a number of exception handlers, onXXX where XXX is the name of the exception being handled. The most specific exception handler implemented will be called. If no method exists for the specific exception being thrown, the exception class hierarchy is traversed with the most specialist exception handler that is implemented being called. Exception handlers are optional. When none are defined, the framework default handler will be executed. This method will only throw an exception if the handler object passed in cannot be used for communication (i.e. it does not implement onSuccess)
    Parameters:
      mappingName - name of the mapping of the service within this form
      parameters - the parameters required by the service
      handler - Object which implements onSuccess and onXXX where XXX is an exception methods
      sync - Currently ignored - all services are asyncronous
      showProcess -
