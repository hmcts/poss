Summary

The database changes are in five parts.  The first applies indexes against the 
columns that are the source of some referential integrity constraints.  The 
second applies indexes against columns used by a series of Oracle Reports used
to compile the Start Of Day reports. The third applies the new MV
required for QBP Case. The fourth script is to update the trigger on USER_ROLE 
table and the last adds a script to create functional index on table wp_output.

Operation

Script(s) run from a SQL*Plus session as the CaseMan user.

Script execution stops immediately at the first sign of an Oracle 
or Operating System error.  

A corresponding Log file is produced, with the naming convention 
<script name>.log (unless otherwise stated).  The file contains 
both progress (at best) and error information (at worst) and will 
reside in the directory where the script was invoked 
(unless otherwise specified).
