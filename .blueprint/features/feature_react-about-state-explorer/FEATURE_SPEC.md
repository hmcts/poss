# Feature Spec: react-about-state-explorer

## Summary
Add an expandable About panel to the State Explorer page that explains how to read the graph, what each visual convention means, and the data sources behind each indicator.

## Problem
The State Explorer page renders a complex interactive graph with coloured nodes, dashed/dotted edges, completeness badges, and WA task badges. Without guidance, users cannot interpret these conventions and may draw incorrect conclusions from incomplete data.

## Solution
Inline `AboutPanel` component (same pattern as `caseman-comparison/page.tsx`) placed directly below the page subtitle area. Collapsed by default. Five sections with h3 headings.

## Sections

| # | Heading | Key content |
|---|---------|-------------|
| 1 | What this page does | Interactive graph of states and transitions; nodes = states, directed edges = transitions |
| 2 | Graph layout | Auto-generated by dagre algorithm; positions are readable but not semantically meaningful |
| 3 | Node colour | Amber=draft/uncertain, green=live/confirmed, dark=terminal end state, muted=low completeness (<50%). Derived from `uncertaintyLevel` in model data |
| 4 | Completeness badge | % of expected model fields populated; low (<50%) means sparse; 0% may mean not yet modelled |
| 5 | WA task badge | Count of WA tasks at this state; colour = worst alignment (red=gap, amber=partial, green=all aligned). Based on wa-mappings.json — incomplete mappings will undercount |
| 6 | Edge style | Solid=user-initiated, dashed=system-triggered, dotted=time-based. Derived from `systemTriggered` flag |

## Content source
Exported string constants from `src/ui-about-state-explorer/index.js`.

## Scope
- `src/ui-about-state-explorer/index.js` — content constants
- `app/state-explorer/page.tsx` — inline AboutPanel component + page header restructure
- `test/feature_react-about-state-explorer.test.js` — content string assertions
